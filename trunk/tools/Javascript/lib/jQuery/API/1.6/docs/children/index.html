.children()

Categories: Traversing > Tree Traversal

.children( [ selector ] ) 返回： jQuery

描述： 获得每个匹配元素集合元素的子元素，选择性筛选的选择器。

  • version added: 1.0.children( [ selector ] )

    selector一个用于匹配元素的选择器字符串。

鉴于一个jQuery对象，表示一个DOM元素的集合，.children()方法允许我们通过在DOM树中对这些元素的直接子元素进行搜索，并且构造一个新的匹配元素的jQuery对象。.find()和.children()方法是相似的，但后者只是针对向下一个级别的DOM树。还要注意的是最喜欢的jQuery方法，.children()不返回文本节点;让所有子元素包括使用文字和注释节点，建议使用.contents()。

该方法选择性地接受同一类型选择器表达式，我们可以传递给$()函数。如果选择供应，将被过滤的元素通过测试它们是否匹配。

考虑一个面页中简单的列表：

<ul class="level-1">
  <li class="item-i">I</li>
  <li class="item-ii">II
    <ul class="level-2">
      <li class="item-a">A</li>
      <li class="item-b">B
        <ul class="level-3">
          <li class="item-1">1</li>
          <li class="item-2">2</li>
          <li class="item-3">3</li>
        </ul>
      </li>
      <li class="item-c">C</li>
    </ul>
  </li>
  <li class="item-iii">III</li>
</ul>

如果我们在2级列表开始，我们可以找到它的子元素：

$('ul.level-2').children().css('background-color', 'red');

此调用的结果是一个落后项目A，B和C红底，由于我们没有提供一个选择的表达，所有的子元素都返回的jQuery对象的一部分。如果我们提供的人，只有三者匹配项目将包括在内。
